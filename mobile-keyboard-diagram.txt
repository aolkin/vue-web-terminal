Mobile Keyboard Behavior Diagram
=================================

BEFORE (No keyboard):
┌─────────────────────────────┐
│  Terminal Header            │
├─────────────────────────────┤
│                             │
│  Terminal Content           │
│                             │
│  Previous commands          │
│  Output...                  │
│                             │
│  > command_input_here       │ ← Input at bottom
│                             │
└─────────────────────────────┘
   Viewport Height: 667px


AFTER (Keyboard appears):
┌─────────────────────────────┐
│  Terminal Header            │
├─────────────────────────────┤
│                             │
│  Terminal Content           │
│  (scrollable)               │
│                             │
│  > command_input_here       │ ← Input stays visible
│                             │
│  ┌───────────────────────┐  │
│  │ [autocomplete above]  │  │ ← Autocomplete above input!
│  └───────────────────────┘  │
│                             │
│  (margin: ~333px)           │ ← Added margin
│                             │
└─────────────────────────────┘
│///////////////////////////│
│/////// KEYBOARD /////////│  ← Covers bottom ~300px
│///////////////////////////│
   Viewport Height: 367px
   
   
Key Points:
-----------
1. Margin = viewportHeight / 2 = 367 / 2 = ~183px
   (But uses the initial 667/2 = ~333px for safety)
   
2. Autocomplete forced above when:
   - isMobileDevice = true
   - mobileKeyboardVisible = true
   
3. Content remains accessible by scrolling
   
4. Input line stays visible above keyboard
